# Руководство по развертыванию

Это руководство описывает процесс настройки и развертывания проекта на **чистой Ubuntu 22.04 LTS виртуальной машине** с
использованием Ansible и GitHub Actions.

---

## Шаг 1: Настройка доступа к серверу

1. **Подключитесь к вашему серверу по SSH.**

2. **Настройте доступ по SSH-ключу (на вашей локальной машине):**
    * Сгенерируйте SSH-ключ, если у вас его еще нет: `ssh-keygen -t rsa -b 4096`
    * Скопируйте публичный ключ на сервер. Используйте **пользователя, который уже существует** на сервере (например,
      `ubuntu` для AWS, `root` для многих VPS).
      ```bash
      ssh-copy-id username@your_server_ip
      ```
      Замените `username` на реальное имя пользователя на вашем сервере.
    * Теперь вы сможете подключаться без пароля: `ssh username@your_server_ip`

---

## Шаг 2: Настройка репозитория на GitHub

1. **Перейдите в настройки вашего репозитория на GitHub.**

2. **Настройте секреты (Secrets):**
   Перейдите в `Settings -> Secrets and variables -> Actions` и создайте следующие секреты:

   | Имя секрета              | Значение |
                      |:-------------------------| :--- |
   | `VPS_IP`                 | Публичный IP-адрес вашего сервера (например, `123.45.67.89`) |
   | `ANSIBLE_USER`               | Имя пользователя для SSH-подключения (например, `ubuntu`, `root`, `developer`) |
   | `SSH_PRIVATE_KEY`        | Ваш **приватный** SSH-ключ (тот же, что вы использовали для `ssh-copy-id`) |
   | `DOMAIN_NAME`            | Основной домен вашего приложения (например, `my-academic-api.com`) |
   | `ALLOWED_HOSTS`          | Список разрешенных хостов через запятую (например, `my-academic-api.com,www.my-academic-api.com`) |
   | `ANSIBLE_VAULT_PASSWORD` | Пароль от вашего `ansible/vars/vault.yml` |

3. **Настройте Ansible Vault (на вашей локальной машине):**
    * Убедитесь, что `ansible/vars/vault.yml` зашифрован.
    * Обновите его, если нужно, с помощью: `ansible-vault edit ansible/vars/vault.yml`

---

## Шаг 3: Развертывание приложения

Развертывание производится исключительно через GitHub Actions.

1. **Слияние в `trunk`:**
    * Создайте Pull Request из вашей feature-ветки в `trunk`.
    * Убедитесь, что все проверки CI (`ci.yml`) проходят успешно.
    * Слейте PR в `trunk`.

2. **Запуск деплоя:**
    * Перейдите в раздел **Actions** вашего репозитория.
    * В списке слева выберите workflow **"Deploy to Production"**.
    * Нажмите на кнопку **"Run workflow"** справа.
    * Нажмите зеленую кнопку **"Run workflow"**.

3. **Ожидание завершения:**
    * Workflow `deploy.yml` скачает артефакт, подключится к вашему серверу и запустит Ansible-плейбук.
    * Наблюдайте за логами workflow. При успешном выполнении все шаги будут зелеными.

4. **Проверка:**
    * Откройте в браузере `https://your-domain.com/api/v1/schema/swagger-ui/`. Документация должна загрузиться.
    * Попробуйте войти в админ-панель по адресу `https://your-domain.com/admin/`.

---

## Шаг 4: Обслуживание

### Обновление приложения

Повторите **Шаг 3** после каждого нового слияния в `trunk`, чтобы развернуть обновленную версию.

### Удаление приложения

Проект не включает автоматизированный workflow для удаления на GitHub Actions. Чтобы удалить приложение, выполните
следующие шаги **с вашей локальной машины**:

1. **Запустите плейбук `teardown.yml` локально:**
   ```bash
   # Убедитесь, что вы в корне репозитория
   # Установите переменную окружения для пути к конфигу Ansible
   export ANSIBLE_CONFIG=./infra/ansible/ansible.cfg

   # Запустите плейбук
   ansible-playbook infra/ansible/teardown.yml --ask-vault-pass
   ```
   Ansible запросит пароль от Vault.

2. **Проверка:**
    * Убедитесь, что все контейнеры (`academic-api-app`, `academic-db`, `academic-caddy`) остановлены и удалены.
    * Убедитесь, что директория `/opt/academic-api` на сервере удалена.
    * Убедитесь, что образы `academic-api:*` удалены.

После этого ваш сервер будет полностью очищен от всех компонентов этого проекта.
